Project: NoKey Wallet
Purpose: A Polkadot-powered proof-of-concept wallet that eliminates seed phrases using Web2-style authentication, device-bound encryption, and social recovery to make Web3 onboarding as easy as Gmail.

Repository: https://github.com/apple101012/NoKeyWalletPolkaDot

## Stack

- Frontend: React 18.3.1 + Vite 5.4.11
- Styling: Tailwind CSS 3.4.15
- Animations: Framer Motion 11.11.17
- Crypto Libraries: @polkadot/util-crypto 13.2.3, crypto-js 4.2.0
- Icons: Lucide React 0.460.0
- Build Tool: Vite with @vitejs/plugin-react

## Runtime / Config

**Environment Variables**: None required for demo (uses mock data)

**Commands**:
- `npm install` - Install dependencies

## User Flow / Interactions

**Main User Journey**:
1. Landing Page → View features and value proposition
2. Click "Create Account" → Open authentication modal
3. Choose auth method → Face ID (simulated) or Email verification
4. Authentication → Simulated biometric or email code entry
5. Success → Wallet created with Polkadot address
6. Dashboard → View balance (2-5 DOT), transactions, parachains
7. Send Transaction → Enter recipient and amount, review, confirm
8. Connect Parachains → Toggle connections to Acala, Moonbeam, Astar, Parallel
9. Recovery → Email or social recovery with trusted contacts
10. Logout → Clear wallet from device

**Key Buttons/Actions**:
- "Create Account" - Generate new wallet
- "Send" - Initiate DOT transfer
- "Receive" - Display wallet address
- "Connect/Disconnect" - Toggle parachain connections
- "Forgot device? Recover" - Open recovery modal
- Copy address icon - Copy wallet address to clipboard
- Logout icon - Clear wallet and return to landing

## Data / Schema

**Wallet Object** (localStorage: 'nokey_wallet', encrypted):
```javascript
{
  address: String,        // Polkadot SS58 address (format 0)
  publicKey: String,      // Hex-encoded public key
  deviceKey: String,      // Hex-encoded seed (encrypted)
  mnemonic: String        // BIP39 mnemonic (never shown to user)
}
```

**Transaction Object** (localStorage: 'nokey_transactions', JSON array):
```javascript
{
  id: Number,
  type: 'sent' | 'received',
  from?: String,          // For received tx
  to?: String,            // For sent tx
  amount: Number,         // DOT amount
  timestamp: Number,      // Unix timestamp
  hash: String            // Mock transaction hash
}
```

**Parachain Data**:
- Acala (△, #E40C5B)
- Moonbeam (◐, #53CBC9)
- Astar (★, #0AE2FF)
- Parallel (◊, #FF4B8B)

**Storage Keys**:
- `nokey_wallet` - Encrypted wallet data
- `nokey_address` - Plain wallet address
- `nokey_balance` - Cached balance
- `nokey_transactions` - Transaction history array

## Known Limitations

- **Demo Only**: Uses mock data, no real blockchain connections
- **Security**: localStorage is not secure for production (should use secure enclave/keychain)
- **Recovery**: Simulated only, real implementation needs threshold cryptography
- **Transactions**: Mock transactions, not broadcast to network
- **Balance**: Deterministic mock balance based on address
- **Authentication**: Simulated biometric/email, not real verification
- **Parachains**: Connection is UI state only, no actual RPC calls
